Font Recognition

A Machine Learning project for UC Davis Data Analytics Bootcamp

Our Purpose

When embarking on this project, we had the intention of helping designers of all expertise levels in mind. With this tool if it were to be built upon, it could serve as both an asset for pro designers looking to identify a font that they came across as well as a learning application for beginner designers.          

The importance of fonts and typefaces is something understood by all designers and consumers alike even if they don't necessarily recognize it. As much as a good design rests on the visual artistic abilities of the designer, it's hard to get a clear and concise message across without words. Fonts act as the voice of design. With the the varying styles of typefaces, the tone of a message can change completely. When that voice matches perfectly with the message, it is seamless to the viewer and usually goes unnoticed. That's what good design is: invisible. Learning how to pair fonts and which ones evoke certain feelings is acquired over time as a designer, but tools like this can help people better understand what differences one should be looking for to begin with.

 

This is why we start off with a basic classification such as serif versus sans serif.   

Here you can help train our machine and see if it can identify whether your font is serif or sans serif! Please make sure your file is either BMP, JPEG, or PNG and less than 100MB.

Our Process 

We began our project by collecting font image data from a Kaggle dataset for sans and sans-serif after looking through Google API and Adobe Typekit API for fonts. Next, we studied some Convolutional Neural Network (CNN) model architecture that we could use to classify such as the number of convolutional layers, pooling layers, number of dense nodes for our binary classification problem. The model architecture we used to solve this classification problem was 3 convolutional layers, with 128 dense nodes in the fully connected layer. In addition, we use applied ‘dropout’ to .5 to help reduce overfitting during training which is where 50% of the neurons are randomly turned off during training. We also used ‘early stopping’ in Keras which allows you to specify the performance measure to monitor and trigger, and once triggered it will end the training process. We focused on ‘val_loss’ with a ‘patience’ of 2, which means number of epochs with no improvement after which training will be stopped. Other hyperparameters included selecting a batch size of 16. After training our model which took 11 epochs the model achieved 95% accuracy during training. When we reviewed the classification report we found the precision performance of when the model predicts yes how often is it correct. Our precision score was 94% for sans serif whereas serif scored at 99% precision.
